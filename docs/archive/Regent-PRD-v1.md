# Regent 产品需求文档 V1

> 状态：编码基线  
> 日期：2026-07-16  
> 原产品名：AgentOS（仅用于历史追溯）

## 1. 产品结论

Regent 接收人的自然语言目标，在明确的约束、资源和授权边界内，自主解释目标、发现能力缺口、组建临时人机组织、创建并运营独立应用，并依据外部结果持续调整计划、能力和组织。

Regent 的核心不是聊天、固定工作流或多 Agent 并发，而是：

> 为目标形成当前最合适的执行组织，并在能力不足时创造能力。

Mission 不设为必填输入。用户只有目标时，Regent 直接工作；目标长期稳定且多个 Goal 出现共同方向时，系统可以提出候选 Mission，由用户选择是否确认。

## 2. 产品边界

产品只有两个一级边界：

```text
regent/
├─ core/   # Regent 自治组织运行内核
└─ apps/   # Regent 创建和运营的独立应用
```

`core` 不包含新闻、童话、订阅、DAU 等业务概念。`apps` 中每个应用拥有独立源码、依赖、数据库、测试、配置和部署生命周期，可以脱离 Core 运行。

场景不是插件、模板或 Core 配置。两个首批场景只是向同一个 Goal 入口提交的真实长期目标。

Challenge、对照实验和研究评估不是用户概念，也不是产品主流程。

## 3. 目标用户

### 3.1 Goal Owner

提交目标，提供资源和边界，处理重大取舍，确认不可逆或高影响行动。

### 3.2 Operator

查看进展、组织、成本、风险和人工待办；可以暂停、恢复、接管和终止。

### 3.3 Capability Provider

AI、工具、服务或人。人可以提供信息、判断、账号、授权、制品和线下行动，不仅是审批节点。

同一用户可以同时承担以上角色。

## 4. 用户输入与系统输出

### 4.1 最小输入

唯一必填项：自然语言 Goal。

可选输入：附件、期限、预算、已有资源、禁止事项、偏好和联系人。用户不需要填写结构化 Challenge、组织图或解决方案。

### 4.2 核心输出

- 目标解释、假设和待确认事项；
- 阶段成果和当前进展；
- 当前组织、成员职责及增减原因；
- 能力缺口及补齐方式；
- 创建或更新的独立 App；
- 外部结果、证据和 KPI 趋势；
- 风险、资源消耗和人工任务；
- 决策、失败与恢复记录。

## 5. 核心闭环

```text
Goal Intake
→ 解释目标并形成可验证阶段结果
→ 识别所需能力与授权
→ 复用、配置、组合、构建或请求能力
→ 形成最小可行组织
→ 规划、执行与协调
→ 独立评价成果
→ 读取外部结果
→ 保持、收缩、替换、扩充组织或重规划
```

Organization 与 Planning 交替迭代：能力池和授权约束计划；计划发现缺口后又会触发组织调整。

## 6. P0 功能需求

### FR-01 Goal Intake

- 接受自然语言 Goal 和可选附件；
- 保存原始输入，不覆盖用户原意；
- 内部生成版本化 GoalSpec；
- 明确区分用户约束、系统推断和未知项；
- 低风险未知项可带假设探索，高风险歧义必须请求确认。

### FR-02 持久目标运行

- Goal、Work、Run、Artifact 和人工等待均可持久化；
- 进程重启后继续运行；
- 支持暂停、恢复、取消、超时和幂等重试；
- 人工等待不占用 Worker。

### FR-03 能力管理

- 从 Goal 和候选计划推导 Capability Requirement；
- 查询已有 AI、Tool、Service 和 Human 能力；
- 区分能力缺失、质量不足、权限不足、资源不足、信息不足和普通执行失败；
- 按 `REUSE → CONFIGURE → COMPOSE → BUILD → HUMAN` 选择补齐方式；
- 新能力必须经过独立验证后才能使用和复用。

### FR-04 自治组织

- 根据能力覆盖、并行收益、权限隔离、上下文隔离、成本和风险形成组织；
- 每个成员必须有职责、输入、输出、权限、预算和退出条件；
- 创建额外 Agent 必须记录理由、预期净收益和独立交付物；
- 子目标冲突时按根 Goal、硬约束、证据质量、可逆性和全局收益裁决；无法裁决时升级人类；
- 支持保持、收缩、替换、扩充、合并、回退单 Agent 和解散。

### FR-05 规划、协调与执行

- 计划是版本化的可执行工作图，不是一次性文本；
- Organization 负责能力、角色、授权和责任；Scheduler 只负责已批准 Work 的时间、优先级、并发、租约和资源分配；
- 共享制品必须声明所有权和合并策略，禁止最后写入覆盖；
- 行动执行前进行策略检查并获得短期 Execution Permit。

### FR-06 App 创建与运营

- Core 可以在 `apps/<app-id>` 创建独立项目；
- 通过通用工作区、代码、构建、测试、制品、部署和观测能力操作 App；
- App 业务数据不进入 Core 数据库；
- App 通过标准 Observation 和 Evidence 返回结果；
- 接入新 App 不得修改 Core 领域模型。

### FR-07 评价、经验与重规划

- 执行者自评不能单独使成果通过；
- 评价保存证据、标准版本和置信度；
- 历史经验记录“上下文—组织—行动—结果—代价”，仅作决策先验，不能覆盖当前硬约束；
- 重规划前先检查约束、依赖、影响范围、可逆性和历史相似结果；
- 连续无可验证进展时停止消耗并请求调整范围、资源或人工能力。

### FR-08 人类协作

- Human Task 是独立持久流程，包含请求原因、期望交付、截止时间、验收标准、无响应后果和替代方案；
- 支持提醒、升级、过期、取消和异步恢复；
- 不假设人工响应时间可预测。

### FR-09 治理与可控性

- 治理是贯穿所有阶段的纵向能力，不是流水线中的单独一层；
- 默认拒绝、最小权限、资源限额和完整审计；
- 支持全局、Goal、Actor 和 Tool 四级停止；
- Agent 可以优化方案，不能自行修改根目标、硬约束、权限上限、KPI 口径和治理规则。

## 7. P1 能力

- 受限 Tool/Skill 自动构建、认证、晋级、撤销和跨 Goal 复用；
- 预览环境、灰度发布和自动回滚；
- 组织方案基于历史经验进行排序；
- 候选 Mission 发现；
- 多 Goal 资源竞争和优先级管理；
- Core 自身候选升级，但必须经过隔离构建、回归验证和人工发布。

## 8. 明确非目标

- 一开始支持任意现实世界目标；
- 强制用户定义 Mission、KPI 树或组织结构；
- 预先设计行业 Agent 和行业工作流；
- 为了展示多 Agent 而强制创建多个 Agent；
- 无限制递归自修改；
- 让 Agent 修改自己的治理、评价标准或生产权限；
- 在 P0 自动执行支付、群发营销、不可逆迁移等高影响动作；
- 把两个 App 的商业成功等同于 Core 技术成立。

## 9. 首批长期运营 Goal

### Goal A

创建并长期运营面向 AI 从业者的产品，覆盖行业动态、理论、前沿进展和实践技巧；第一阶段达到 100 个有效付费用户。

### Goal B

创建并长期运营面向成年人的短篇童话网站，持续生产高传播潜力的原创故事；第一阶段达到 10,000 有效 DAU。

两者以相同自然语言 Goal API 进入 Regent。产品形态、技术栈、功能和运营策略由 Regent 在边界内探索，不预先固化。付费和 DAU 必须来自外部可验证数据，Core 无权修改口径或原始记录。

## 10. P0 产品验收

1. 用户仅提交一个自然语言 Goal 即可启动；
2. Core 在没有任何 App 时独立启动并完成虚拟 Goal；
3. 系统能解释为何使用一个或多个 Agent，并能主动收缩组织；
4. 能识别至少一种真实能力缺口，通过构建 Tool 或请求人类补齐并独立验证；
5. Worker 被终止后 Goal 可从持久状态恢复，外部副作用不重复；
6. 能创建一个完全独立的 App 项目，完成构建、测试并生成可审阅制品；
7. 更换为第二个 Goal 时不修改 Core 领域模型；
8. 人工任务可等待、过期和恢复，不阻塞运行线程；
9. 子目标冲突产生可追溯裁决，而不是静默覆盖；
10. 所有高风险行动被拒绝或进入明确审批流程。

## 11. 成功指标

### Core 技术指标

- Goal 恢复成功率 100%；
- 已声明外部副作用重复执行率 0；
- 关键状态与审计记录覆盖率 100%；
- 未授权高风险操作成功数 0；
- 新 App 接入导致 Core 业务模型改动数 0；
- 新能力均有独立验证证据；
- 无进展循环能够在预算阈值内停止。

### 商业结果指标

100 个有效付费用户和 10,000 有效 DAU 是两个 App 的长期 Goal 指标，用于检验 Regent 持续运营能力，不作为 P0 编码完成的前置条件。

## 12. P0 完成定义

Regent 能从普通自然语言 Goal 出发，在无预置场景流程的情况下，形成可解释的最小组织，补齐至少一个能力缺口，在独立 `apps` 目录创建可运行产品候选，并通过持久状态、外部证据和治理边界持续推进目标。
