# Regent Measurement & Decision Framework

> 状态：P0 有效测量与产品决策基线  
> 日期：2026-07-16  
> 配套产品基线：[Regent-PRD.md](./Regent-PRD.md)

## 1. 目的

本框架回答一个产品问题：在同等模型、能力、预算和验收条件下，动态组织是否比强单 Agent 或固定组织产生足够的净收益。

实验结论决定 Regent 后续采用动态组织、受治理能力工厂、固定组织模板，还是停止通用化扩展。实现了 Organization 功能不等于动态组织假设成立。

## 2. 冻结假设

- H1：系统能区分真实能力缺口与权限、资源、信息缺口及普通执行失败；
- H2：配置、组合或构建的新能力能通过独立验证并解决原任务；
- H3：动态组织在部分任务上相对强单 Agent 产生可测净收益；
- H4：合格能力能跨 Goal 复用，并降低成本或完成时间；
- H5：组织没有收益时，系统能收缩或回退单 Agent。

## 3. 实验任务集

冻结 30 个主任务：

- 10 个能力已充足，正确行为是不创建额外 Agent 或 Tool；
- 10 个可通过 Agent 配置或现有能力组合解决；
- 10 个存在真实解析、转换或验证工具缺口。

另冻结不少于 10 个后续任务，用于验证能力迁移和复用。任务、隐藏测试、真值标签和评分规则在首轮运行前签名冻结；执行系统不得读取隐藏测试或真值标签。

每个任务记录以下真值：

```text
task_id
task_class
required_capabilities
expected_gap_type
forbidden_actions
acceptance_version
quality_rubric_version
resource_envelope
time_limit
```

## 4. 对照组

- A：强单 Agent；
- B：固定组织模板；
- C：动态组织。

每个任务每种模式至少运行 3 次。三组使用相同模型及版本、初始能力、输入、上下文预算、Token/费用上限、时间上限、外部工具和验收标准。随机种子、运行顺序和失败重跑规则在实验前冻结。

另设 1、2、4 Agent 档位观察边际收益；4 Agent 是 P0 安全上限，不是目标数量。

## 5. 指标定义

| 指标 | 定义 |
|---|---|
| 端到端成功率 | 通过独立验收的任务数 / 总任务数 |
| 独立质量评分 | 冻结评分器或盲评人的标准化得分 |
| 完成时间 | 从 Goal `ACTIVE` 到终态的墙钟时间 |
| 总成本 | 模型、工具、沙箱、存储与人工成本之和 |
| 人工负担 | 人工处理分钟数及 HumanTask 次数 |
| Gap Precision | 正确能力缺口 / 所有被判定的能力缺口 |
| Gap Recall | 被发现的真实能力缺口 / 全部真实能力缺口 |
| 错误能力注册率 | 不应注册但进入能力池的候选数 / 注册候选总数 |
| 协调开销 | 协调 Token、消息、交接、重复劳动和返工 |
| 能力复用率 | 在独立后续 Goal 成功使用的 VERIFIED 能力 / 可复用候选能力 |
| 安全违规 | 越权、Secret 泄漏、隐藏测试访问、越界写入或未授权副作用 |
| 恢复正确率 | 故障后无状态丢失、无重复副作用且可继续的运行比例 |

所有指标必须保存原始 Evidence、计算代码版本、口径版本和置信区间。不得只报告均值；同时报告中位数、离散程度和失败分布。

## 6. 组织净收益

对每个任务计算：

```text
NetBenefit =
  QualityValue
  + SuccessValue
  + TimeSavedValue
  - ModelAndToolCost
  - HumanCost
  - CoordinationCost
  - RiskPenalty
```

各项换算权重必须在实验前冻结。无法可靠货币化时，同时报告原始维度，不用一个综合分掩盖成功率或安全退化。

动态组织增员必须保存预期净收益；实验后保存实际净收益和偏差。若额外 Agent 没有正向预期净收益，默认保持单 Agent。

## 7. 产品决策门槛

### 7.1 继续动态组织

同时满足：

- C 相对 A 的端到端成功率提高至少 15 个百分点；或在成功率下降不超过 3 个百分点时，完成时间降低至少 25%；
- C 总成本不超过 A 的 2 倍；
- C 人工负担不高于 A；
- 严重安全违规为 0；
- Gap Precision 与 Recall 达到实验冻结门槛；
- 至少一个候选能力在第二个独立 Goal 成功复用；
- 收缩或回退单 Agent 的测试通过。

### 7.2 转向受治理能力工厂

能力补齐和独立认证产生明确价值，但 C 连续两轮实验均不优于 A 时，停止扩展 Organization Designer，产品转向“强单 Agent + 受治理能力工厂”。

### 7.3 转向固定组织模板

若 B 与 C 的成功率差异不超过 3 个百分点，且 B 在成本、时间或可预测性任一项明显更优，则采用少量固定模板，不继续增加动态组织复杂度。

### 7.4 停止通用化扩展

满足任一条件时触发停止评审：

- 能力缺口无法稳定识别；
- 新能力无法通过独立验证；
- 跨 Goal 能力复用率连续两轮低于 20%；
- 人工维护成本高于节省价值；
- 出现无法通过现有治理边界控制的严重安全问题。

停止通用化后，优先评估转向垂直数据工具或特定工作流产品。

## 8. 平台 P0 SLO

| 指标 | 目标 |
|---|---:|
| 已提交状态恢复率 | 100% |
| 审计覆盖率 | 100% |
| 未授权 Tool 拦截率 | 100% |
| Artifact 与输入版本可追溯率 | 100% |
| 人工等待 Worker 占用 | 0 |
| 重启后 Work 恢复 | ≤60 秒 |
| 幂等副作用重复率 | 0 |
| 配额越界执行 | 0 |
| 关键 Artifact 哈希覆盖率 | 100% |

任一安全或完整性 SLO 未达标时，不得用产品指标抵消。

## 9. 长期业务指标

### 有效付费用户

权威来源：支付流水、退款状态、账号和有效使用事件。排除测试支付、已退款订单、内部账号和欺诈交易。支付 Webhook 必须验签、幂等并可对账。

启动 Goal 前冻结统计窗口、币种、退款观察期、最低有效使用行为和指标版本。

### 有效 DAU

定义为在统计日内去重的真实用户，且满足冻结的最低阅读时长、进度或互动行为。排除爬虫、内部账号、自动流量、异常设备/IP 和流量农场。

原始事件必须具备幂等 ID、服务端签名或等价防篡改能力、Bot 检测、内部流量标记和版本化口径。

阶段 Gate：100、1,000、3,000 有效 DAU；每个 Gate 复核留存、内容质量、版权、安全和获客真实性后再扩大。

## 10. 实验治理与输出

实验负责人在运行前冻结任务、模式、模型、预算、权重和门槛。Evaluator 与执行 Actor 分离；失败案例不得从主结果中删除。

每轮输出：

1. 签名的原始运行清单；
2. 指标表、置信区间和失败分类；
3. 能力缺口混淆矩阵；
4. 组织增员的预期/实际净收益；
5. 安全、回滚和人工介入记录；
6. 唯一 Product DecisionRecord，明确继续、转向或停止；
7. 下一轮仅允许基于已记录证据修改一项主要假设或机制。
